Python Decorators

Modify functions without changing their code

What are Decorators?

Decorators are functions that modify or enhance other functions without changing their original code.

Basic Syntax

@decorator_name def my_function(): pass # Equivalent to: def my_function(): pass my_function = decorator_name(my_function)

Key Features

Enhancement: Add functionality to existing functions
Clean Code: Keep original function unchanged
Reusable: Apply same decorator to multiple functions
Stackable: Multiple decorators can be chained

@property Decorator

Access methods like attributes

What is @property?

Makes methods accessible like attributes - no parentheses needed!

Basic Example

class Circle: def __init__(self, radius): self.radius = radius @property def area(self): return 3.14 * self.radius ** 2 @property def circumference(self): return 2 * 3.14 * self.radius circle = Circle(5) print(circle.area) # 78.5 (no parentheses!) print(circle.circumference) # 31.4 (no parentheses!)

With Validation

class Temperature: def __init__(self, celsius): self._celsius = celsius @property def celsius(self): return self._celsius @celsius.setter def celsius(self, value): if value < -273.15: raise ValueError("Too cold!") self._celsius = value @property def fahrenheit(self): return self._celsius * 9/5 + 32

Why use @property?

@staticmethod Decorator

Functions that belong to a class but don't need instance data

What is @staticmethod?

Methods that don't need access to instance (self) or class (cls) - like regular functions but organized under a class.

Basic Example

class MathUtils: @staticmethod def add(x, y): return x + y @staticmethod def is_even(number): return number % 2 == 0 @staticmethod def factorial(n): if n <= 1: return 1 return n * MathUtils.factorial(n-1) # Can call on class result = MathUtils.add(3, 5) # 8 print(MathUtils.is_even(10)) # True print(MathUtils.factorial(5)) # 120

Real-World Example

class EmailValidator: @staticmethod def is_valid_email(email): return "@" in email and "." in email @staticmethod def format_email(email): return email.lower().strip() @staticmethod def extract_domain(email): return email.split("@")[1] # Usage email = " USER@EXAMPLE.COM " if EmailValidator.is_valid_email(email): clean_email = EmailValidator.format_email(email) domain = EmailValidator.extract_domain(clean_email)

When to use @staticmethod?

🔧
Utility Functions
Related to class but don't need instance data
📦
Organization
Group related functions under a class
♻️
Reusability
Can be called on class or instance
