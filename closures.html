Python Closures

Functions that "remember" their environment

What are Closures?

A technique for functions to access variables in a limited scope

A closure is a function that has access to variables in its outer (enclosing) scope even after the outer function has finished executing.

Think of it as a way to parametrize functions - creating specialized functions from general ones.

Mathematical Example

Creating parametrized functions

We want to create functions: g₁₀(x) = 10x, g₅(x) = 5x, g₂₀(x) = 20x

In other words: y ↦ g_y(x) = x × y

General Function

g(x, y) = x × y

→

Parametrized

g₁₀(x) = 10x

→

Specialized

g₅(x) = 5x

→

More Specialized

g₂₀(x) = 20x

Python Implementation

Using closures to create parametrized functions

def create_multiplier(y): # This is the outer function def multiplier(x): # This is the inner function (closure) return x * y # 'y' is captured from outer scope return multiplier # Return the inner function # Create specialized functions g_10 = create_multiplier(10) # g₁₀(x) = 10x g_5 = create_multiplier(5) # g₅(x) = 5x g_20 = create_multiplier(20) # g₂₀(x) = 20x # Use the parametrized functions print(g_10(3)) # 30 (3 × 10) print(g_5(4)) # 20 (4 × 5) print(g_20(2)) # 40 (2 × 20)

How Closures Work

Step-by-step breakdown

1. Outer Function Called

g_10 = create_multiplier(10) # y = 10 is stored in the closure

2. Inner Function Returned

# g_10 is now a function that "remembers" y=10 # It's like: def g_10(x): return x * 10

3. Closure Called

result = g_10(3) # Uses the captured y=10: 3 * 10 = 30

4. Variable Persists

# Even though create_multiplier() finished, # g_10 still has access to y=10

Real-World Example

Creating specialized validators

def create_validator(min_value, max_value): def validate(value): return min_value <= value <= max_value return validate # Create specialized validators age_validator = create_validator(0, 120) grade_validator = create_validator(0, 100) temperature_validator = create_validator(-50, 50) # Use the validators print(age_validator(25)) # True print(grade_validator(85)) # True print(temperature_validator(75)) # False

Key Benefits

Why closures are powerful

🎯 Function Parametrization

Create specialized functions from general ones without repeating code

🔒 Data Encapsulation

Variables are "private" to the closure - only accessible through the returned function

⚡ State Persistence

Functions can "remember" values even after the outer function completes

🔄 Flexible Design

Create multiple specialized functions with different configurations
