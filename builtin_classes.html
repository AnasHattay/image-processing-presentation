Built-in Classes in Python

Ready-to-use classes with powerful methods

What are Built-in Classes?

Python comes with many pre-defined classes that you can use immediately

Why Use Built-in Classes?

  • No need to create from scratch
  • Optimized and well-tested
  • Rich set of built-in methods
  • Consistent across Python programs

Common Built-in Classes

String Class (str)

Powerful text manipulation with many built-in methods

upper()
lower()
strip()
split()
replace()
text = " Hello, World! " print(text.strip().upper()) # "HELLO, WORLD!" print(text.split(", ")) # [" Hello", "World! "]" print(text.replace("World", "Python")) # " Hello, Python! "

List Class (list)

Dynamic arrays with extensive manipulation methods

append()
insert()
remove()
pop()
sort()
numbers = [3, 1, 4, 1, 5] numbers.append(9) # [3, 1, 4, 1, 5, 9] numbers.sort() # [1, 1, 3, 4, 5, 9] print(numbers.count(1)) # 2 numbers.reverse() # [9, 5, 4, 3, 1, 1]

Dictionary Class (dict)

Key-value storage with efficient lookup methods

keys()
values()
get()
pop()
update()
person = {"name": "Alice", "age": 25} print(person.get("name", "Unknown")) # "Alice" person.update({"city": "Paris"}) # Add new key print(list(person.keys())) # ["name", "age", "city"]

Set Class (set)

Unique elements with mathematical set operations

add()
remove()
union()
intersection()
difference()
set1 = {1, 2, 3, 4} set2 = {3, 4, 5, 6} print(set1.union(set2)) # {1, 2, 3, 4, 5, 6} print(set1.intersection(set2)) # {3, 4} print(set1.difference(set2)) # {1, 2}

Common Magic Methods

Special methods available across built-in classes

__str__() User-friendly string
str(123) # "123" str([1,2,3]) # "[1, 2, 3]"
__add__() Addition operator
5 + 3 # calls (5).__add__(3) "Hello" + " World" # calls "Hello".__add__(" World")

Arithmetic Methods

__sub__() Subtraction (-)
5 - 3 # 2 10 - 4 # 6
__len__() Length (len)
len("Hello") # 5 len([1,2,3,4]) # 4

🔮 How Magic Methods Work

When you use operators or functions, Python automatically calls the corresponding magic method!

Examples:
print(obj) # calls obj.__str__() len(obj) # calls obj.__len__() obj + other # calls obj.__add__(other) obj == other # calls obj.__eq__(other) obj[key] # calls obj.__getitem__(key)
Behind the Scenes:
# When you write: a + b # Python actually does: a.__add__(b) # Same for print: print(a) # becomes a.__str__()

What if We Want to Create Our Own Types?

Built-in classes are great, but what about custom data types?

The Limitation

Built-in classes are fixed! You can't modify their behavior or create new types with custom logic.

# What if we want a "Student" type? # Or a "BankAccount" type? # Or a "Car" type? # Built-in classes can't help us here!

The Solution

Object-Oriented Programming! Create your own classes with custom attributes and methods.

# We can create our own types! class Student: def __init__(self, name, grade): self.name = name self.grade = grade # Now we have a custom "Student" type!
