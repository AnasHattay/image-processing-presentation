Pass by Value vs Pass by Reference

Note: The following discussion is theoretical on programming. Python does not let you directly manage memory like C/C++, but you still must be aware of some concepts.

Understanding Assignment: a = b

In programming, there are 2 ways to understand assignments. There are two ways you can interpret the action a = b.

Pass by Value

When you "pass by value", you are saying:

a.value = b.value

If b is an integer with value 5, then a will be assigned an integer with value 5.

Memory Impact: a and b will occupy different spaces in memory (i.e., they will occupy twice the memory).

VS

Pass by Reference

When you "pass by reference", you are saying:

a.reference = b.reference

Here, reference refers to the place in memory a variable lives. This is somewhat like pointers in C/C++.

Memory Impact: a and b point to the same location in memory. No new memory is created.

Why Does It Matter?

Let us work with a practical case:

Scenario:

  • Assume b.value = 5 and b.reference = 0x00
  • You copy the reference to b, to a: a.reference = b.reference
  • Now, let us assume you do b.value = 10

Result: Since a and b refer (or point to) the same place in memory, when you try to access a.value, you will get 10.

Going Back to Python

When you pass by reference, you don't create new memory. In this case a and b will point to the same location in memory.

In practice, this means that when you access the value of a and the value of b, you will get the same value (because you will look at the same place in memory).
