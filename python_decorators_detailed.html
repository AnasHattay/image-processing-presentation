Decorator Syntax

Two ways to use decorators

üí° Key Point

Both methods do exactly the same thing! The @ symbol is just a cleaner way to write it.

Method 1: @ Symbol (Recommended)

@my_decorator def my_function(): return "Hello!"

Method 2: Manual (Same Result)

def my_function(): return "Hello!" my_function = my_decorator(my_function)

Decorator Example

Measuring and printing a function's execution time

üéØ Goal

We want to measure how long a function takes to run and print the elapsed time, without changing the original function.

Step 1: Create the Decorator

def timer(func): # This is our decorator function def wrapper(): import time start_time = time.time() result = func() # Call the original function end_time = time.time() print(f"Function {func.__name__} took {end_time - start_time:.4f} seconds") return result return wrapper # Return the wrapper function

üí° What's happening: We create a function that takes another function and returns a new function (wrapper) that adds extra behavior.

Step 2: Use the Decorator

@timer def calculate_sum(): total = 0 for i in range(1, 1001): total += i return total

üí° What's happening: The @ symbol applies our timer decorator to the calculate_sum function. Now calculate_sum() is actually the wrapper function!

Step 3: Call the Function

result = calculate_sum() # Output: # Function calculate_sum took 0.0002 seconds # result = 500500

üí° What's happening: When we call calculate_sum(), we're actually calling the wrapper function, which measures and prints the execution time of our original function.
