Object-Oriented Programming Fundamentals

What are Classes?

A class is a user-defined data type that encapsulates data and functions together. It serves as a template for creating objects with specific attributes and behaviors.

Data Encapsulation: Classes group related data (attributes) and functions (methods) into a single unit
Code Reusability: One class definition can be used to create multiple objects with the same structure
Object Creation: Classes define the structure and behavior that all instances of that class will have

Why Use Classes?

Modularity: Break complex problems into manageable pieces
Reusability: Create multiple objects from the same class
Encapsulation: Keep data and methods together, control access
Organization: Structure code in a logical, maintainable way

Class Example: Vector

class Vector: def __init__(self, x, y): self.x = x self.y = y def magnitude(self): return (self.x**2 + self.y**2)**0.5 def add(self, other): return Vector(self.x + other.x, self.y + other.y) # Creating objects (instances) v1 = Vector(3, 4) v2 = Vector(1, 2) # Using the objects print(v1.magnitude()) # 5.0 v3 = v1.add(v2) # Vector(4, 6)

What is 'self'?

'self' is a reference to the current instance of the class. It's like saying "this specific object" when you're inside a method.

# When you call: my_vector.magnitude() # Python automatically passes 'my_vector' as 'self' def magnitude(self): # self = my_vector return (self.x**2 + self.y**2)**0.5 # self.x = my_vector.x

Why 'self' is Required

🎯 Access Instance Data

Methods need to access the object's attributes (name, age)

🔄 Modify State

Methods can change the object's attributes

🔗 Call Other Methods

One method can call another method on the same object

How 'self' Works

class Vector: def __init__(self, x, y): self.x = x # Store in THIS object self.y = y # Store in THIS object def magnitude(self): return (self.x**2 + self.y**2)**0.5 # Access THIS object's x, y def add(self, other): return Vector(self.x + other.x, self.y + other.y) # Creating objects v1 = Vector(3, 4) # self = v1 v2 = Vector(1, 2) # self = v2 v1.magnitude() # self = v1, returns 5.0 v2.magnitude() # self = v2, returns 2.24

Important Rules About 'self'

✅ Always Required

Every method must have 'self' as the first parameter

✅ Python Adds It

You don't pass 'self' when calling methods - Python does it automatically

✅ Name Can Vary

You can use any name, but 'self' is the convention

✅ Access Everything

Use 'self' to access any attribute or method of the object

Additional important things about classes

__init__ Constructor

Special method called when creating a new object. It receives arguments for setting attributes.

def __init__(self, name, age): self.name = name self.age = age

Method vs Function

Methods belong to classes and always have 'self' as first parameter. Functions are standalone.

# Method (inside class) def magnitude(self): return (self.x**2 + self.y**2)**0.5 # Function (outside class) def distance(p1, p2): return ((p2.x - p1.x)**2 + (p2.y - p1.y)**2)**0.5
