What are Getters & Setters?

Getters and Setters are methods that control how attributes are accessed and modified, providing data validation and encapsulation.

Getter: Method to read/access an attribute
Setter: Method to modify/change an attribute
Validation: Control what values are allowed
Encapsulation: Hide internal implementation

Why Use Getters & Setters?

Data Validation: Ensure only valid values are stored
Computed Values: Calculate attributes on-the-fly
Logging: Track when attributes are accessed/changed
Flexibility: Change implementation without breaking code

Basic Example: Temperature Class

Using @property for getters and setters

class Temperature: def __init__(self, celsius=0): self._celsius = celsius # Private attribute @property def celsius(self): """Getter for celsius temperature""" return self._celsius @celsius.setter def celsius(self, value): """Setter for celsius temperature""" if value < -273.15: raise ValueError("Temperature below absolute zero!") self._celsius = value @property def fahrenheit(self): """Computed property for fahrenheit""" return self._celsius * 9/5 + 32 # Usage temp = Temperature(25) print(temp.celsius) # 25 (getter) print(temp.fahrenheit) # 77.0 (computed) temp.celsius = 30 # setter print(temp.fahrenheit) # 86.0

Advanced Example: Person Class

Multiple getters and setters with validation

class Person: def __init__(self, name, age): self._name = name self._age = age @property def name(self): return self._name.title() # Always capitalize @name.setter def name(self, value): if not isinstance(value, str): raise TypeError("Name must be a string") if len(value.strip()) == 0: raise ValueError("Name cannot be empty") self._name = value.strip() @property def age(self): return self._age @age.setter def age(self, value): if not isinstance(value, int): raise TypeError("Age must be an integer") if value < 0 or value > 150: raise ValueError("Age must be between 0 and 150") self._age = value @property def is_adult(self): return self._age >= 18
