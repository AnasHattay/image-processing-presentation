Mutability & Nested Data Structures

Since lists store elements by their references, if we change the element in the list it will change everywhere else.

Example: List containing a List

# Original list original_list = [1, 2, 3] # Create a list containing the original list container_list = [original_list, 'other_data'] print("Before:", container_list)

Output:

Before: [[1, 2, 3], 'other_data']

Key Question: What happens when we modify original_list?

# Modify the original list original_list.append(4) print("After:", container_list)

Output:

After: [[1, 2, 3, 4], 'other_data']

Result: The change appears in container_list because lists are mutable and stored by reference!

Contrast: Immutable vs Mutable

Tuple containing a List (Mixed Mutability)

# Tuple containing a mutable list my_tuple = ([1, 2, 3], 'immutable_string') print("Before:", my_tuple) # Can't modify the tuple itself # my_tuple[0] = [4, 5, 6] # This would raise an error! # But can modify the list INSIDE the tuple my_tuple[0].append(4) print("After:", my_tuple)

Output:

Before: ([1, 2, 3], 'immutable_string') After: ([1, 2, 3, 4], 'immutable_string')

Important: The tuple structure is immutable, but the contents of mutable objects within it can still change!
